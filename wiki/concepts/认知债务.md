# 认知债务

## 定义

认知债务（Cognitive Debt）是指团队或组织在软件开发过程中，由于**规范缺失、文档不完善、架构混乱**等原因累积的"认知差距"。

类似于技术债务，但更致命：
- **技术债务**：代码质量问题，最终需要还
- **认知债务**：需求理解、架构决策、业务认知的偏差，会严重影响 AI Agent 的协作效率

## 认知债务的来源

1. **需求不清晰**：没有文档化，用户和 Agent 对"什么是正确答案"理解不一致
2. **规范缺失**：没有约定代码格式、文档结构，Agent 产出难以统一
3. **上下文丢失**：关键决策只存在于某人的脑子中，新人/新 Agent 无法理解

## Agentic Engineering 如何解决认知债务

Claude Code 等工具的思路：
- 通过 **Superpowers** 规定文档格式（如 SPEC、PRD）
- 在 plan 模式下不断完善和打磨文档
- 修改时自动更新所有相关文档

这种方式天然建立了**常态化弥合认知债务的机制**。

## 解决思路

1. **显式化**：把脑子里的知识写到文档中
2. **结构化**：使用固定格式（SPEC、PRD、API 文档）
3. **可追溯**：决策过程完整记录，便于回顾
4. **自动化**：Agent 修改时自动同步更新相关文档
