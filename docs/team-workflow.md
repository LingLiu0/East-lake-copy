# East-lake 团队协作指南

> 基于 GitHub 的 AI 知识库团队协作方案

---

## 协作架构

```
┌─────────────┐     git push      ┌─────────────┐     GitHub Actions      ┌─────────────┐
│  成员 A     │ ───────────────→  │   GitHub    │ ────────────────────→  │  知识库     │
│  (本地)     │                   │   仓库      │                        │  (自动更新)  │
└─────────────┘                   └─────────────┘                         └─────────────┘
                                        ↑
┌─────────────┐     git push           │
│  成员 B     │ ───────────────────────┘
│  (本地)     │
└─────────────┘
```

---

## 两种协作模式

### 模式 A：协作者（推荐）

适用于小团队，直接拥有仓库写入权限

```
1. 仓库管理员添加成员：Settings → Collaborators
2. 成员克隆仓库：git clone
3. 本地添加资料：raw/clippings/ 或 raw/articles/
4. 推送：git push
5. GitHub Actions 自动编译 + 优化
6. 其他成员 git pull 同步
```

### 模式 B：Fork 贡献

适用于开源项目或外部贡献者

```
1. Fork 仓库到个人账户
2. 克隆自己的 Fork
3. 添加资料并推送到自己的 Fork
4. 发起 Pull Request
5. 管理员审核合并
6. 合并后自动触发编译
```

## 配置步骤

### 1. 仓库管理员操作

```markdown
1. 创建 GitHub 仓库
2. Settings → Secrets and variables → Actions

添加 Secrets:
- API_KEY      → 大模型 API Key
- API_BASE     → API 端点地址
- MODEL        → 模型名称
```

### 2. 添加协作者

```markdown
1. Settings → Collaborators
2. Add people → 输入用户名/邮箱
3. 发送邀请
```

### 3. 协作者配置

```bash
# 克隆仓库
git clone https://github.com/团队名/East-lake.git
cd East-lake

# 安装依赖（可选）
pip install requests beautifulsoup4

# 添加资料
# - 方式 A：Obsidian + Web Clipper
# - 方式 B：直接复制文件到 raw/

# 推送
git add raw/
git commit -m "feat: 添加新文章"
git push
```

## 团队使用场景

### 场景 1：每周例会知识沉淀

```
1. 成员 A 将会议记录放入 raw/articles/
2. 推送 → 自动编译成摘要+概念
3. 成员 B 下周可通过提问回顾会议内容
```

### 场景 2：行业研究分工

```
成员 A：负责 AI 行业新闻 → raw/clippings/AI新闻/
成员 B：负责竞品分析 → raw/articles/竞品/
成员 C：负责技术论文 → raw/articles/论文/

→ 各自推送 → 自动合并成统一知识库
```

### 场景 3：外部投稿

```
外部贡献者：Fork → 添加资料 → 发起 PR
管理员：审核 → 合并 → 自动编译
```

## 常见问题

**Q: 如何知道谁添加了什么资料？**
A: 每周周报会显示贡献者统计，或查看 git log

**Q: 多人同时推送会冲突吗？**
A: GitHub Actions 会依次处理，推送前先 git pull 同步

**Q: 没有 API Key 还能用吗？**
A: 本地搜索不需要，编译功能正常，但自动优化需要 API

**Q: 不想用默认的模型？**
A: 在 GitHub Secrets 中修改 MODEL 参数
